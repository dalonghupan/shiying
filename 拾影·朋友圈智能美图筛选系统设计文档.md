## 1\. 需求规格说明

### 1\.1 产品愿景

打造轻量化桌面图片筛选工具，支持常规算法快速筛选与 AI 大模型精细化筛选双模式，离线处理保障隐私，帮助用户高效挑出适配朋友圈发布的高清优质照片。

### 1\.2 核心用户需求

#### 基础操作需求

1. 支持选取本地多级图片目录，批量加载千级数量图片

2. 可视化预览图片，支持手动勾选调整筛选结果

3. 一键导出精选图片至自定义文件夹

4. 界面风格柔和美观，操作逻辑简单易懂

#### 图片筛选核心需求

1. **基础筛选**：识别图片清晰度、常规构图质量，自动打分排序

2. **AI 增强筛选**：可配置大模型 Agent，多维判别光影、氛围感、人像姿态、场景美感

3. **模式自由切换**：用户按需选择筛选方式，无模型配置自动降级兜底

4. 可自定义筛选严格阈值，灵活把控精选标准

#### 非功能需求

1. 性能：1000 张图片加载≤3s，常规筛选≤10s，AI 推理分片控速无卡顿

2. 兼容：适配 Windows、macOS 系统，支持 JPG/PNG/WEBP/HEIC 格式

3. 稳定：大批量处理无闪退、内存无溢出，模型异常不中断业务流程

4. 安全：图片全程本地流转，禁止公网上传，保护个人隐私

### 1\.3 系统边界

- 输入：本地目录图片文件、大模型连接配置参数

- 处理：本地图像算法分析 / 本地大模型 Agent 智能研判、结果汇总排序

- 输出：可视化精选图片列表、导出本地图片文件

- 范围排除：不含修图、滤镜编辑功能；仅支持本地模型服务对接

### 1\.4 约束条件

1. 默认离线基础算法运行，AI 模式需用户手动配置模型参数启用

2. 单目录最大支持 5000 张图片批量处理

3. 打包后为独立程序，终端设备无需预装 Python 运行环境

4. 模型调用仅内网本地交互，不向外网传输原图素材

## 2\. 技术解决方案

### 2\.0 开发环境配置（正式规范）

**开发操作系统**：MacOS 系统（适配主流 Ventura / Sonoma 及以上版本）

**开发Python版本**：Python 3\.11\.x（稳定正式版，禁止Beta测试版）

**环境特性说明**：

- 基于MacOS原生终端开发，支持Homebrew环境管理，无系统兼容冲突

- Python3\.10为项目指定唯一标准开发版本，兼容所有UI、图像处理、模型对接依赖库

- 开发环境与最终跨平台打包环境隔离，可正向编译出Windows、MacOS双端独立程序

- 环境纯净无冗余依赖，规避版本冲突、库兼容报错问题，保障项目稳定性

### 2\.1 开发技术栈

- 主体语言：Python 3\.10

- UI 框架：PyQt6

- 图像处理：OpenCV、Pillow、Numpy

- 模型交互：轻量 HTTP 客户端、本地模型调用 SDK

- 打包工具：PyInstaller / Nuitka，跨平台生成独立可执行程序

### 2\.2 架构分层设计

采用五层模块化分层架构，低耦合易扩展

1. **界面展示层**：目录选择、模式切换、图片预览、结果导出、状态提示

2. **业务逻辑层**：目录遍历、任务调度、筛选结果管理、操作事件响应

3. **AI Agent 调度层**：模式判定、模型任务分发、评分格式统一、异常自动降级

4. **图像分析层**：清晰度检测、构图特征计算、多维度图像特征提取

5. **文件数据层**：图片读写、缩略图缓存、筛选文件导出管理

### 2\.3 双筛选逻辑

#### 模式一：本地基础算法（默认启用）

1. 清晰度判定：拉普拉斯方差算法识别模糊、失焦画面

2. 构图评分：检测主体位置、对称度、三分线构图合规性

3. 综合计分：清晰度占比 60%\+ 构图占比 40%，按分数排序筛选

#### 模式二：大模型 Agent 增强筛选

1. 用户填写本地模型接口地址、端口、密钥完成配置

2. 系统压缩生成缩略图提交本地模型服务

3. 模型多维评估画质、光影、氛围感、朋友圈适配度并输出 0\-100 分值

4. 调度层统一评分标准，完成图片排序精选

### 2\.4 模型接入方式

1. 离线私有化模型：本地部署多模态视觉模型，程序内部直接推理

2. 本地 API 服务：对接内网模型接口，仅本地网络数据交互

### 2\.5 筛选模式对比

|筛选模式|核心优势|适用场景|资源消耗|
|---|---|---|---|
|本地算法|响应迅速、无额外依赖、稳定性高|大批量快速初选|低|
|大模型 Agent|审美判别精准、氛围感识别强|精细化精选配图|中高|

### 2\.6 语言选型佐证

1. Python 图像处理、多模态模型调用生态成熟，快速实现双筛选逻辑

2. PyQt6 可高效开发高颜值桌面界面，贴合女性使用审美

3. 支持多线程异步任务，满足千张图片处理性能要求

4. 打包后脱离运行环境，跨平台分发使用便捷

5. Go、 Rust 图像与 UI 生态薄弱，开发周期长，不适配本业务场景

## 3\. 系统模块设计

### 3\.1 目录图片加载模块

遍历过滤图片文件，异步加载生成缩略图缓存，实现可视区域懒加载

### 3\.2 基础算法筛选模块

完成清晰度、构图计算，生成基础质量评分与筛选列表

### 3\.3 AI 模型配置模块

提供参数录入、模式开关、服务连通性检测、配置重置功能

### 3\.4 Agent 调度容错模块

自动识别运行模式，分发分析任务，模型异常自动切回基础算法

### 3\.5 可视化预览模块

卡片式展示图片，标注评分来源与分数，支持手动选择调整

### 3\.6 图片导出模块

批量拷贝精选图片，创建专属文件夹，展示导出进度状态

### 3\.7 全局性能管控模块

内存自动回收、并发任务限流，避免大批量处理卡顿溢出

## 5\. 性能优化方案

1. 线程隔离：UI 线程负责界面渲染，后台线程处理图片分析与模型推理

2. 缓存策略：缩略图本地缓存，重复访问无需重复读取磁盘文件

3. 分片处理：AI 模式拆分批量任务，控制单次并发数量，降低资源占用

4. 内存管控：及时释放闲置图片资源，整机内存占用控制合理范围

5. 轻量化处理：仅传输缩略图用于模型分析，缩减数据传输耗时

### 5\.1 性能达标指标

- 1000 张图片加载耗时 ≤2s

- 全量基础筛选耗时 ≤8s

- 界面滚动流畅，帧率稳定 60 帧

- 程序运行内存占用 ≤800MB
