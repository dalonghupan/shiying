# 拾影·开发方案

**策略**：垂直切片法 — 先跑通完整链路，再逐层叠加 AI、UI、性能优化、打包
**总工期**：10 天
**技术栈**：Python 3.11 + PyQt6 + OpenCV + Pillow + NumPy + httpx

---

```
shiying/
```

## 3. 各阶段实现要点

### P1 脚手架（Day 1）

- `main.py`：QApplication 启动，加载 QSS 样式表
- `main_window.py`：QMainWindow 基础框架，划分顶部栏/侧边栏/主区/底部栏四个区域
- `requirements.txt`：PyQt6, opencv-python, Pillow, numpy, requests, httpx
- `config.py`：全局常量（支持格式列表、默认阈值、缓存路径、缩略图尺寸）

**验收标准**：运行 main.py 弹出空白主窗口，四个区域占位正确

### P2 核心链路（Day 2-3）

**图片加载模块 `core/image_loader.py`**
- `os.walk` 遍历目录，过滤 JPG/PNG/WEBP/HEIC
- QThreadPool 异步加载，生成 256x256 缩略图
- PyQt 信号通知 UI 刷新（每批 20 张）

**基础算法筛选 `core/algorithm_filter.py`**
- 清晰度：`cv2.Laplacian(gray, cv2.CV_64F).var()` 拉普拉斯方差
- 构图：三分线检测（主体是否在 1/3 交叉点附近）、对称度计算
- 综合评分：清晰度 60% + 构图 40%，归一化到 0-100

**图片预览 `ui/preview_panel.py`**
- QGridLayout + QLabel 卡片式网格
- 每张卡片：缩略图 + 评分 badge + 勾选框
- 点击卡片弹出大图预览

**导出模块 `core/exporter.py`**
- `shutil.copy2` 批量拷贝已勾选图片到目标文件夹
- QProgressBar 实时显示进度
- 导出完成提示

**验收标准**：选择文件夹 → 图片加载显示 → 自动打分排序 → 勾选 → 导出到指定目录

### P3 AI Agent（Day 4-5）

**模型配置 `ui/sidebar.py` 配置区**
- QFormLayout：接口地址、端口、API Key 输入框
- 连通性测试按钮：发一个简单请求验证模型可达
- 配置保存/重置

**Agent 调度 `core/ai_agent.py`**
- httpx.AsyncClient 调用本地模型 API
- 发送 base64 缩略图，解析返回 0-100 评分
- 支持多维评分：画质、光影、氛围感、朋友圈适配度

**统一评分 `core/scorer.py`**
- 接口统一：无论算法还是 AI，都返回 `{score: int, source: "algorithm"|"ai", details: dict}`
- 调度层根据当前模式选择评分来源

**容错降级**
- 请求超时 10s / 连接失败 / 返回格式异常 → 自动切回基础算法
- UI 状态栏提示"AI 模型异常，已降级为基础算法模式"

**验收标准**：配置模型地址 → 切换 AI 模式 → 图片获得 AI 评分 → 断开模型 → 自动降级并提示

### P4 UI 完善（Day 6-7）

**主题样式 `assets/styles.qss`**
- 主色调：淡粉 #F8E8EE + 浅紫 #E8D5E3
- 背景：白色 #FFFFFF，边框浅灰 #E0E0E0
- 按钮 hover：粉色渐变
- 字体：系统默认，14px 正文，16px 标题

**侧边栏完善**
- 阈值滑块：QSlider 0-100，实时调整筛选标准
- 统计面板：总图片数、已筛选数、平均分、最高分
- 模式切换：QRadioButton 基础/AI 模式

**顶部工具栏**
- 目录选择：QPushButton + QLabel 显示当前路径
- 筛选按钮：开始筛选 / 重新筛选
- 导出按钮：导出已选图片
- 模式指示：当前模式图标 + 文字

**底部状态栏**
- QProgressBar 任务进度
- QLabel 运行状态文字（加载中/筛选中/就绪/错误）

**悬浮预览**
- 鼠标悬浮在缩略图上 → 弹出 512x512 预览
- 优质图片（分数 > 80）添加金色边框标识

**验收标准**：界面柔和美观，操作流畅，状态反馈及时

### P5 性能优化（Day 8）

**线程隔离**
- UI 线程只做渲染和事件响应
- 图像分析全部走 QThreadPool + QRunnable
- AI 推理走独立线程池，避免阻塞 UI

**缩略图缓存**
- 缓存 key：`hashlib.md5(absolute_path + str(thumbnail_size)).hexdigest()`
- 缓存目录：`~/.shiying/cache/thumbnails/`
- 首次加载生成缓存，后续直接读取

**分片处理**
- AI 模式：每批 10 张图片，批间 sleep 50ms
- 避免内存尖峰，控制并发数

**内存管控**
- 图片加载后释放原始大图引用
- 缓存使用 LRU 策略，超过 500MB 自动清理
- 定期调用 `gc.collect()`

**验收标准**：1000 张图片加载 ≤2s，基础筛选 ≤8s，UI 滚动 60fps，内存 ≤800MB

### P6 打包发布（Day 9-10）

**macOS 打包**
- PyInstaller `--onefile --windowed --name 拾影`
- 生成 .app，可用 create-dmg 打包为 .dmg
- 签名：ad-hoc 签名避免 macOS 安全警告

**Windows 打包**
- PyInstaller `--onefile --windowed --name 拾影`
- 生成 .exe，附带 assets 目录
- 可选 Inno Setup 生成安装程序

**验收标准**：双平台独立程序可运行，无需预装 Python，功能完整

## 5. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| HEIC 格式兼容 | macOS 照片多为 HEIC | 使用 pillow-heif 插件支持 |
| AI 模型接口不统一 | 不同模型 API 格式不同 | scorer.py 做适配层，支持 OpenAI 兼容格式 |
| 大批量内存溢出 | 5000 张图片 OOM | 分片加载 + LRU 缓存 + 及时释放 |
| PyQt6 跨平台样式差异 | macOS/Windows 渲染不一致 | QSS 为主，平台特定微调 |
| 打包体积过大 | 依赖库多，包体积大 | Nuitka 编译优化，排除不必要模块 |
