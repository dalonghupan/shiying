## 修复记录

### 2026-05-26

**#1 PyQt6 启动报错**
- 原因：直接 `python3 main.py` 使用系统 Python，未安装 PyQt6
- 修复：使用虚拟环境 `./venv/bin/python main.py` 或先 `source venv/bin/activate` 再 `python main.py`

**#2 AI测试连接崩溃**
- 文件：`ui/main_window.py`
- 原因：`_test_ai_connection` 未处理 httpx 异常，连接失败时程序崩溃
- 修复：添加 try-catch 包裹异步调用，异常时返回 False 并提示用户，finally 中关闭 agent

**#3 AI配置默认显示**
- 文件：`ui/sidebar.py`
- 原因：AI 模型配置 GroupBox 默认可见，容易误触发
- 修复：默认隐藏 `model_group.hide()`，监听模式切换信号，选择 AI 模式时 `model_group.setVisible(True)`

**#4 图片加载卡在11%**
- 文件：`core/image_loader.py`
- 原因：批次触发逻辑仅在 `_loaded_count % THUMBNAIL_LOAD_BATCH == 0` 时推进，加载失败的图片不计数，导致批次卡住
- 修复：改用 `_processed_count` 统计已处理数（成功+失败），错误也触发 `_check_batch_complete()` 推进批次

**#5 图片加载数量不对（161→159）**
- 文件：`utils/cache.py`
- 原因：2 张 RGBA 模式的 PNG 图片在保存缩略图时，JPEG 不支持 RGBA 模式，抛出 `cannot write mode RGBA as JPEG`
- 修复：`create_thumbnail` 中检测 RGBA/P/LA 模式，先转 RGB 再保存

**#6 导出按钮不可点击**
- 文件：`ui/main_window.py`
- 原因：`_on_all_loaded` 中 `enable_export(False)` 禁用了导出按钮，筛选完成后未重新启用
- 修复：在 `_on_score_result` 和 AI 筛选完成时调用 `enable_export(True)`

**#7 切换模式后评分来源被覆盖**
- 文件：`ui/main_window.py`
- 原因：`_on_score_result` 根据 `self._current_mode` 动态判断 source，切换模式后所有评分都显示新模式的来源
- 修复：新增 `_score_sources` 字典，评分时记录实际来源，切换模式不影响已评分图片的来源显示

**#8 AI地址为空测试连接崩溃**
- 文件：`ui/main_window.py`
- 原因：仅检查空字符串，未校验 URL 格式，无效 URL 导致 httpx 异常
- 修复：添加 `api_url.startswith(("http://", "https://"))` 校验，不合法时提示用户

**#9 DeepSeek AI 筛选崩溃**
- 文件：`core/ai_agent.py`, `ui/main_window.py`, `ui/sidebar.py`
- 原因：模型名硬编码为 `"default"`，DeepSeek 需要具体模型名；AI 响应可能包含 markdown 代码块，直接 json.loads 会失败；连续失败时无熔断机制
- 修复：
  - 侧边栏新增「模型名称」输入框，支持 `deepseek-chat` / `deepseek-vl` 等
  - AI 响应 JSON 解析增加容错（支持 markdown 代码块、文本中提取 JSON）
  - 连续 3 次失败自动停止筛选并弹窗提示，避免程序崩溃

**#10 日志系统缺失**
- 文件：`utils/logger.py`（新建）, `config.py`, `main.py`, `ui/main_window.py`, `core/ai_agent.py`, `core/image_loader.py`, `core/exporter.py`, `utils/thread_manager.py`, `utils/cache.py`
- 原因：项目无任何日志输出，异常被静默吞掉，排查问题困难
- 修复：
  - 新增 `utils/logger.py`，使用 `RotatingFileHandler`（5MB，3 备份）写入 `~/.shiying/logs/shiying.log`
  - `config.py` 新增 `LOG_DIR` 常量
  - `main.py` 入口处调用 `setup_logging()`
  - 6 个模块共 12 处添加 `logger.error()` / `logger.warning()` / `logger.info()` / `logger.debug()` 调用

**#11 API 地址格式错误导致 404**
- 文件：`ui/main_window.py`, `core/ai_agent.py`, `ui/sidebar.py`
- 原因：用户输入 `https://api.deepseek.com/anthropic` 等错误地址，请求发到不存在的端点返回 404
- 修复：
  - `main_window.py` 添加 URL 格式校验，要求包含 `/chat/completions` 或 `/v1/`
  - `ai_agent.py` 的 `test_connection` 方法优化 URL 转换逻辑，支持多种 URL 格式
  - `sidebar.py` 添加 tooltip 提示常见 API 地址格式

**#12 纯文本模型发送图片请求导致 400**
- 文件：`core/ai_agent.py`, `ui/sidebar.py`
- 原因：`deepseek-chat` 是纯文本模型，不支持图片输入，但代码发送了包含 `image_url` 的请求
- 修复：
  - `ai_agent.py` 添加 `_is_vision_model()` 方法，通过关键词（vl/vision/gpt-4o 等）判断模型是否支持图片
  - 非视觉模型调用 `score_image()` 时抛出 `ValueError`，提示用户使用视觉模型
  - `sidebar.py` 更新 placeholder 和 tooltip，明确提示需要视觉模型

**#13 DeepSeek API 不支持图片分析**
- 文件：`core/ai_agent.py`, `ui/sidebar.py`
- 原因：DeepSeek API 仅支持文本内容，不支持 `image_url` 类型。API 返回错误：`unknown variant 'image_url', expected 'text'`
- 修复：
  - `ai_agent.py` 添加 `TEXT_ONLY_PREFIXES` 集合，将 `deepseek` 标记为不支持图片
  - `_is_vision_model()` 方法先检查是否为已知纯文本提供商
  - 错误信息明确说明 DeepSeek 不支持图片，并建议使用 OpenAI/Anthropic/Google
  - `sidebar.py` 更新 placeholder 和 tooltip，说明 DeepSeek 不支持图片分析
  - 添加 API 错误响应体日志，便于排查
